# 📁 types/

**Objetivo:** Definições e interfaces TypeScript que não pertencem exclusivamente a um contexto de negócio, mas são usadas em múltiplos lugares.

**Exemplos:**
- Tipagens de dados analíticos abstratos (`ChartData.ts`).
- Tipos de usuários administradores e privilégios base.

**Regras:**
- Interfaces ultra-específicas de um contexto devem residir na pasta `types/` de suas respectivas features.
